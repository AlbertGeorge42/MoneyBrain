# 退款与手续费/优惠券功能 Spec

## Why
当前系统无法手动记录退款，且所有交易类型都缺少手续费和优惠券支持。需要：
1. 支持手动记录退款功能
2. 为收入、支出、转账、退款添加手续费和优惠券字段
3. 正确计算账户余额（考虑手续费和优惠券）

## What Changes
- 修改 Prisma schema，添加 `fee` 和 `coupon` 字段
- 修改后端交易创建/更新逻辑，正确处理手续费和优惠券
- 修改前端交易表单，支持退款类型和手续费/优惠券输入
- 修改导入逻辑，支持手续费和优惠券字段

## Impact
- Affected code:
  - `backend/prisma/schema.prisma` - 添加字段
  - `backend/src/routes/transaction.ts` - 修改余额计算逻辑
  - `frontend/src/pages/Transactions.tsx` - 添加表单字段
  - `backend/src/routes/data.ts` - 修改导入逻辑

## ADDED Requirements

### Requirement: 退款记录功能
系统 SHALL 支持手动记录退款：
- 退款类型选择
- **必须关联原交易记录**
- 退款金额可自由填写（不限制为原交易金额）
- 选择退款账户（可能与原交易账户不同）

### Requirement: 手续费支持
系统 SHALL 为所有交易类型支持手续费：
- 手续费字段（可选，默认为0）
- 手续费从交易金额中额外收取
- 正确更新账户余额

### Requirement: 优惠券支持
系统 SHALL 为所有交易类型支持优惠券：
- 优惠券字段（可选，默认为0）
- 优惠券减少实际支出/增加实际收入
- 正确更新账户余额

## MODIFIED Requirements

### Requirement: 余额计算逻辑
系统 SHALL 在交易时正确计算账户余额：

**收入交易**：
- 账户余额 += 金额 - 手续费 + 优惠券
- 实际到账 = 金额 - 手续费 + 优惠券

**支出交易**：
- 账户余额 -= 金额 + 手续费 - 优惠券
- 实际支出 = 金额 + 手续费 - 优惠券

**转账交易**：
- 转出账户余额 -= 金额 + 手续费 - 优惠券
- 转入账户余额 += 金额

**退款交易**：
- 退款账户余额 += 金额
