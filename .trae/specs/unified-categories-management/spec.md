# 统一分类管理与现金流量表优化 Spec

## Why
当前分类管理分散在多个页面（账户管理、设置），用户体验不统一。需要提供一个统一的分类管理页面，同时优化现金流量表的数据来源配置。

## What Changes
- 创建统一的分类管理页面，整合资产/负债分类和收入/支出分类管理
- 支持一级、二级分类的层级管理
- 添加现金流量表配置功能，定义哪些账户属于现金及现金等价物
- **BREAKING**: 移除分散的分类管理入口，统一到新的分类管理页面

## Impact
- Affected specs: 分类管理模块、现金流量表模块
- Affected code: 
  - 前端: 新增 CategoriesManage 页面，修改 Settings、Accounts 页面
  - 后端: 可能需要添加现金流配置相关 API

## ADDED Requirements

### Requirement: 统一分类管理页面
系统应提供一个统一的分类管理页面，支持管理以下分类类型：
- 资产分类（一级、二级）
- 负债分类（一级、二级）
- 收入分类（一级、二级）
- 支出分类（一级、二级）

#### Scenario: 创建一级分类
- **WHEN** 用户在分类管理页面选择分类类型并点击"新增一级分类"
- **THEN** 系统显示创建表单，用户输入名称和图标后保存

#### Scenario: 创建二级分类
- **WHEN** 用户选择一个一级分类并点击"新增二级分类"
- **THEN** 系统显示创建表单，自动关联父分类，用户输入名称和图标后保存

#### Scenario: 编辑分类
- **WHEN** 用户点击分类的编辑按钮
- **THEN** 系统显示编辑表单，用户可修改名称和图标

#### Scenario: 删除分类
- **WHEN** 用户点击分类的删除按钮且该分类无子分类、无关联数据
- **THEN** 系统删除该分类并刷新列表
- **WHEN** 分类有关联数据
- **THEN** 系统提示无法删除

### Requirement: 现金流量表配置
系统应支持配置现金及现金等价物账户，用于生成现金流量表。

#### Scenario: 配置现金账户
- **WHEN** 用户在分类管理页面标记某个账户分类为"现金等价物"
- **THEN** 该分类下的所有账户将纳入现金流量表计算

#### Scenario: 查看现金流量表
- **WHEN** 用户查看现金流量表
- **THEN** 系统仅统计标记为现金等价物的账户交易

## MODIFIED Requirements

### Requirement: 账户分类模型
AccountCategory 模型需新增字段：
- `isCashEquivalent`: Boolean - 标识是否为现金等价物（用于现金流量表）

### Requirement: 页面导航
主导航菜单需调整：
- 新增"分类管理"菜单项
- 移除设置页面中的分类管理标签页
- 移除账户管理页面中的分类管理功能

## REMOVED Requirements

### Requirement: 分散的分类管理入口
**Reason**: 统一到新的分类管理页面
**Migration**: 将设置页面和账户管理页面的分类管理功能迁移到新的统一页面
