# 重构桑基图绘制逻辑规格

## Why
当前桑基图节点排列存在混杂问题：收入分类和非现金账户在来源侧混在一起，支出分类和非现金账户在去向侧混在一起，不够清晰直观。

## What Changes
重新设计桑基图节点排列逻辑，确保分类清晰分离。

### 新桑基图结构

```
现金来源              现金账户              现金去向
┌─────────┐         ┌────────┐         ┌─────────┐
│ 收入分类 │         │        │         │ 支出分类 │
│  工资    │────────▶│        │────────▶│  三餐   │
│  奖金    │────────▶│ 支付宝 │────────▶│  交通   │
│  利息    │────────▶│        │────────▶│  购物   │
├─────────┤         ├────────┤         ├─────────┤
│非现金账户│         │ 微信   │         │非现金账户│
│  花呗    │────────▶│        │────────▶│  储蓄卡  │
│  信用卡  │────────▶└────────┘────────▶│  基金   │
└─────────┘                            └─────────┘
```

### 节点定义与排列顺序

| 层级 | 节点类型 | 排列顺序 | 数据来源 |
|------|---------|---------|---------|
| 左侧-上 | 收入分类 | 按金额降序 | 收入交易分类 |
| 左侧-下 | 非现金账户 | 按金额降序 | 转账转入的来源账户 |
| 中间 | 现金账户 | 按金额降序 | 标记为现金等价物的账户 |
| 右侧-上 | 支出分类 | 按金额降序 | 支出交易分类 |
| 右侧-下 | 非现金账户 | 按金额降序 | 转账转出的目标账户 |

### 节点分类标识

```typescript
interface SankeyNode {
  name: string
  category: 'income_category' | 'non_cash_source' | 'cash' | 'expense_category' | 'non_cash_target'
  depth: number  // 用于控制节点水平位置
}
```

### 节点分类说明

| category | 说明 | 颜色建议 |
|----------|------|---------|
| income_category | 收入分类节点 | 绿色系 |
| non_cash_source | 非现金账户（来源） | 青色系 |
| cash | 现金账户节点 | 蓝色系 |
| expense_category | 支出分类节点 | 红色系 |
| non_cash_target | 非现金账户（去向） | 橙色系 |

## Impact

### 后端修改
- `backend/src/routes/report.ts`: 修改桑基图节点生成逻辑，区分收入分类和非现金账户

### 前端修改
- `frontend/src/components/charts/SankeyChart.tsx`: 更新节点颜色配置，支持新的分类标识

## ADDED Requirements

### Requirement: 节点分类标识
后端API应返回更细粒度的节点分类：
- `income_category`: 收入分类节点
- `non_cash_source`: 非现金账户（来源侧）
- `cash`: 现金账户节点
- `expense_category`: 支出分类节点
- `non_cash_target`: 非现金账户（去向侧）

### Requirement: 节点排序
节点应按金额降序排列，确保金额大的节点显示在上方。

### Requirement: 任意时间段支持
桑基图应支持任意时间段的绘制，包括：
- 单日
- 单周
- 单月
- 多月
- 自定义日期范围

## MODIFIED Requirements

### Requirement: 节点颜色配置
前端应根据新的分类标识配置颜色：
- 收入分类：绿色 (#52c41a)
- 非现金账户（来源）：青色 (#13c2c2)
- 现金账户：蓝色 (#1890ff)
- 支出分类：红色 (#ff4d4f)
- 非现金账户（去向）：橙色 (#fa8c16)
