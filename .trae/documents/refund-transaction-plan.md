# 退款记录功能计划

## ✅ 已完成

### 问题分析

从钱迹CSV数据中发现退款记录：

```csv
ID,时间,分类,二级分类,类型,金额,币种,账户1,账户2,备注,...,关联账单
qj17725972263072840,2026-03-04 12:06:59,餐饮,三餐,退款,2.87,CNY,微信,,,,,,...,qj1772593594255147741
qj1772593594255147741,2026-03-04 11:06:22,餐饮,三餐,支出,11.87,CNY,云闪付,,,,,,...,
```

### 退款记录特点
1. 类型为"退款"
2. 有"关联账单"字段，指向原支出记录ID
3. 金额为正数（退回的金额）
4. **退款账户可能与原支出账户不同**（如上例：原支出用云闪付，退款到微信）

### 实现方案

1. **导入逻辑**：
   - 第一遍：解析所有行数据，分离退款和非退款记录
   - 第二遍：先处理非退款记录，建立 ID 映射表
   - 第三遍：处理退款记录，查找关联的原交易ID

2. **导出逻辑**：
   - 支持 `refund` 类型导出
   - 导出关联账单ID
