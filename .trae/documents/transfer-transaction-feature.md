# 转账交易功能与页面重构

## 目标
1. 增加转账交易记录的支持
2. 修改收支记录页面的名称和布局，更切合新增功能后的子页面

## 需求分析

### 转账交易特点
- 涉及两个账户：转出账户和转入账户
- 不影响总资产（只是资产在不同账户间转移）
- 不需要收支分类
- 转账记录在报表中单独展示

### 页面重构需求
- 页面名称从"收支记录"改为"交易记录"
- 新增"记转账"按钮
- 交易列表支持显示转账类型记录
- 页面布局优化，区分收支和转账

## 实施步骤

### Step 1: 数据库模型修改
- Transaction 模型添加 `type = 'transfer'` 支持
- 添加 `toAccountId` 字段（转账目标账户）
- 添加 `toAccount` 关联

### Step 2: 后端 API 修改
- 修改交易创建接口，支持转账类型
- 转账时同时更新两个账户余额
- 删除转账时回滚两个账户余额
- 交易列表支持按类型筛选

### Step 3: 前端页面重构
- 页面标题改为"交易记录"
- 新增"记转账"按钮
- 转账表单（选择转出账户、转入账户、金额、日期、备注）
- 交易列表显示转账记录
- 统计卡片调整（转账不影响收支统计）

## 详细设计

### 1. 数据库模型修改

```prisma
model Transaction {
  id           String    @id @default(uuid())
  type         String    // 'income' | 'expense' | 'transfer'
  amount       Decimal
  date         DateTime
  note         String?
  isAdjustment Boolean   @default(false)
  accountId    String    // 转出账户/收入账户/支出账户
  account      Account   @relation("TransactionFrom", fields: [accountId], references: [id])
  toAccountId  String?   // 转账目标账户
  toAccount    Account?  @relation("TransactionTo", fields: [toAccountId], references: [id])
  categoryId   String?   // 转账时可为空
  category     Category? @relation(fields: [categoryId], references: [id])
  createdAt    DateTime  @default(now())

  @@map("transactions")
}
```

### 2. 转账交易逻辑

**创建转账**：
```
1. 验证转出账户和转入账户不同
2. 验证转出账户余额充足
3. 扣减转出账户余额
4. 增加转入账户余额
5. 创建转账记录
```

**删除转账**：
```
1. 回滚转出账户余额（增加）
2. 回滚转入账户余额（减少）
3. 删除转账记录
```
