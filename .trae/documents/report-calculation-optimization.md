# 报表计算逻辑优化与平账功能

## 目标
1. 优化报表计算逻辑，根据账户初始值和收支记录共同计算
2. 手动矫正资产负债表后自动生成平账收支记录
3. 在收入支出表和现金流量表中添加期初/期末数据
4. 完善现金流量表账户信息配置

## 需求分析

### 1. 资产负债表计算逻辑
**改进方案**：
- 展示当月月初（1日 00:00:00）的资产负债状况
- 计算逻辑：初始余额 + 月初之前的所有收支 = 月初余额
- 手动矫正时自动生成平账收支记录

### 2. 收入支出表增强
**新增内容**：
- 期初资产：所选期间开始时的总资产
- 期末资产：所选期间结束时的总资产
- 资产变动：期末 - 期初

### 3. 现金流量表增强
**新增内容**：
- 期初现金：所选期间开始时的现金等价物总额
- 期末现金：所选期间结束时的现金等价物总额
- 现金变动核对：期末 - 期初 = 净现金流

### 4. 账户管理增强
**新增字段**：
- 账户支持标记现金流量活动类型（经营/投资/筹资）

## 实施步骤

### Step 1: 数据库模型修改
- Account 模型添加 `cashFlowType` 字段
- Transaction 模型添加 `isAdjustment` 字段
- 创建系统分类"平账调整"

### Step 2: 后端 API 重构
- 资产负债表：计算月初数据，支持平账
- 收入支出表：添加期初/期末资产
- 现金流量表：添加期初/期末现金
- 平账 API：自动生成平账记录

### Step 3: 前端页面修改
- Reports.tsx：添加期初/期末展示
- AccountCategoryModal.tsx：账户支持活动类型
