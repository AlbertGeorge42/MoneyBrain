# 重构账户分类和账户列表逻辑

## 目标
将账户分类和账户列表整合到同一个流程中，用户可以在统一的树形结构视图中完成账户的新增、分类、编辑、删除操作。

## 当前问题分析

### 现有实现
- 两个独立的标签页：账户分类、账户列表
- 分类和账户管理分离，用户需要切换标签页
- 操作流程不连贯

### 用户需求
- 在同一个流程下完成所有操作
- 减少页面切换，提升操作效率

## 设计方案

### 采用树形结构整合
将账户作为分类的子节点，在同一个树形视图中展示：
```
📁 资产分类
  ├─ 📁 现金及现金等价物 [现金等价物开关]
  │   ├─ 📁 银行存款
  │   │   ├─ 💰 工商银行储蓄卡 (¥12,345.67) [编辑] [删除]
  │   │   └─ 💰 招商银行储蓄卡 (¥8,888.88) [编辑] [删除]
  │   └─ 💰 微信钱包 (¥123.45) [编辑] [删除]
  ├─ 📁 投资账户
  │   └─ 💰 证券账户 (¥50,000.00) [编辑] [删除]
  └─ [+ 添加子分类] [+ 添加账户]

📁 负债分类
  ├─ 📁 信用卡
  │   └─ 💰 招商信用卡 (-¥5,000.00) [编辑] [删除]
  └─ [+ 添加子分类] [+ 添加账户]
```

### 交互设计
1. **一级分类行**：显示分类名称、现金等价物开关、添加子分类按钮、添加账户按钮、编辑按钮、删除按钮
2. **二级分类行**：显示分类名称、添加账户按钮、编辑按钮、删除按钮
3. **账户行**：显示账户图标、名称、余额、编辑按钮、删除按钮
4. **顶部工具栏**：新增资产分类、新增负债分类按钮
5. **汇总统计**：顶部显示总资产、总负债、净资产

## 实施步骤

### Step 1: 重构 AccountCategoryModal 组件
- 移除独立的标签页结构
- 采用统一的树形表格展示
- 分类和账户混合显示

### Step 2: 实现树形数据结构
- 构建包含分类和账户的混合树形数据
- 分类节点可包含子分类和账户
- 账户作为叶子节点

### Step 3: 实现统一的操作逻辑
- 在分类行提供添加子分类和添加账户按钮
- 账户创建时自动关联父分类
- 编辑和删除操作根据节点类型执行

### Step 4: 优化用户体验
- 顶部显示汇总统计
- 区分分类和账户的视觉样式
- 提供快速操作入口

## 详细实现

### 1. 树形数据结构
```typescript
interface TreeNode {
  id: string
  key: string
  name: string
  icon: string
  type: 'category' | 'account'
  nodeType: 'asset' | 'liability'
  balance?: number  // 仅账户有
  isCashEquivalent?: boolean  // 仅一级资产分类有
  parentId: string | null
  children?: TreeNode[]
}
```

### 2. 表格列定义
- **图标**：分类用📁，账户用💰或自定义
- **名称**：分类名称或账户名称
- **余额**：仅账户显示，分类不显示
- **现金等价物**：仅一级资产分类显示开关
- **操作**：根据节点类型显示不同操作按钮

### 3. 操作按钮设计
- **一级分类行**：[添加子分类] [添加账户] [编辑] [删除]
- **二级分类行**：[添加账户] [编辑] [删除]
- **账户行**：[编辑] [删除]

### 4. 表单优化
- 新增账户时自动填充所属分类
- 新增子分类时自动填充父分类和类型
- 减少用户输入项
